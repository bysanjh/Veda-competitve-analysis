04
Key UX learnings for Veda
From Chani
A strong author voice builds trust faster than any feature
Chani Nicholas's editorial voice is the product's moat — users trust it because it sounds like a person, not a system. Veda's character voice needs the same intentionality.
Voice design
From Chani
"Go back in time" — past readings as a personal archive
Archiving past content gives the product a memory even when AI memory doesn't exist. Veda's storybook does this better — but the instinct to save the journey is right.
Archive mechanic
From Co–Star
Social comparison drives organic acquisition
Comparing charts with friends is Co–Star's entire growth loop. Veda's relationship layer + shareable compatibility cards should serve the same function — but with depth.
Growth mechanic
From Moonly
The locked-but-visible paywall tease converts
Showing all 6 life area scores locked behind birth details is highly effective. User sees exactly what they're missing. One CTA to unlock. Veda should adopt this for the daily energy screen.
Monetisation
From Moonly
Structured daily habit with multiple content types
Lesson + article + meditation = 20 mins, 10M users. The variety of format within a daily structure keeps engagement high. Veda's oracle + sense queue + calendar could follow this pattern.
Habit loop
From Replika
Memory review consent screen — show your work
"Here's what I noticed — choose what to keep." Radical transparency builds extraordinary trust. Users don't feel tracked — they feel heard. Directly steal this for Veda's sense queue UX.
Trust mechanic
From Replika
The diary written from the companion's perspective
"Sam and I had a conversation about the beach." The AI narrates the relationship. Turns chat logs into meaningful events. Veda's storybook does this at a higher level — monthly not daily.
Narrative memory
From Pi
Ask one good question and genuinely wait
Pi's conversational style: one question, then silence. Never rushes to advice. Never fires three prompts. Veda should adopt this — question-first, reading-second. Listening is the product.
Conversation design
From ChatGPT
Memory as transparent, editable settings — but make it beautiful
ChatGPT shows memory as a bullet list in settings. The transparency is right. The experience is soulless. Veda's three-tier memory system does this with meaning — same principle, entirely different feeling.
Memory UX
From all competitors
The 1/20 counter kills trust before it starts
Every competitor either hides the paywall or shows it after value is delivered. Veda's current "1/20" on load is the worst first impression in the category. Fix immediately.
Urgent fix
From all competitors
Nobody connects chart + calendar + people + story
Every competitor owns one pillar. Co–Star: chart. Moonly: content. Replika: memory. Pi: conversation. Veda is the only product that can hold all four. That's the entire positioning argument.
White space
From all competitors
Embedded audience is an uncopiable advantage
Co–Star, Chani, Moonly all built standalone apps and paid for users. Veda has 400 organic daily chats from a 10M+ monthly audience already seeking spiritual guidance. Nobody can buy this.
Strategic moat

Veda knows all of it."
